VPM XML 구조 및 역할 분석

1. VPM XML 전체 계층 트리 구조

  • <vpmapp> (Root: VPM 설정 전체)
    • 기본 정보 및 설정
      • <vpmxml-info> XML 버전 정보
      • <enforcement-point> 정책 적용 활성/비활성
    • 글로벌 리스트 정의
      • <Known-File-Ext> ➔ <File-Ext> 파일 확장자 정의
      • <Known-Recip-Lists> ➔ <Recip-List> 이메일 수신자 명단
    • <conditionObjects> (정책 조건 객체 - 핵심 정의 영역)
      • 네트워크/인프라: <ipobject>, <host-port>, <protocol>
      • 웹/앱 카테고리:
        • <vpm-cat> ➔ <node>, <child>
        • <categorylist4> ➔ <sel>, <i>, <excl>
        • <app-url> ➔ <selected-applications>
      • 요청/응답 상세: <a-url>, <op-url>, <user-agent-list>, <hdr-obj>
      • 보안 및 제어: <ssl-fwd-prxy>, <svr-cert>, <file-download>
      • 복합 객체: <comb-obj> ➔ <c-l-1>, <c-l-2> (여러 조건을 하나로 결합)
    • <configObjects> 환경 설정 객체 (예: dns-rdns)
    • <layers> (정책 실행 계층)
      • <layer> (개별 정책 테이블)
        • <name> / <numRows>
        • <rowItem> (개별 규칙)
          • <colItem>, <colItemV2> (규칙의 각 열: Source, Dest, Action 등)
      • <raw-cpl> 직접 작성한 CPL(Content Policy Language) 코드

2. 주요 항목별 상세 역할 표

구분 XML 태그 (항목) 상세 역할 설명
시스템/버전 <vpmxml-info>, <enforcement-point> XML의 생성 정보와 이 정책이 현재 장비에 적용 중인지를 나타냅니다.
파일/이메일 <Known-File-Ext>, <Known-Recip-Lists> 자주 사용하는 파일 확장자 그룹과 경고 메시지를 보낼 이메일 주소 그룹을 정의합니다.
URL/카테고리 <vpm-cat>, <categorylist4>, <a-url> 접속 허용/차단 대상이 되는 URL 주소, 카테고리, 그리고 그 안의 세부 동작을 정의합니다.
애플리케이션 <app-url>, <selected-applications> Facebook, YouTube 등 특정 웹 앱 기반의 정밀한 제어 조건을 설정합니다.
네트워크 조건 <ipobject>, <host-port>, <protocol> IP 주소, 포트 번호, 사용 프로토콜 및 정책이 적용될 시간 범위를 설정합니다.
HTTP 헤더/동작 <hdr-obj>, <user-agent-list> HTTP 헤더 값(Referer 등)을 검사/수정하고 브라우저 종류에 따른 필터링을 수행합니다.
SSL/인증서 <ssl-fwd-prxy>, <svr-cert-valdn> 암호화된 트래픽(HTTPS)의 복호화 검사 여부와 서버 인증서의 유효성을 검사합니다.
파일 제어 <file-download>, <appar-data-type> 실제 다운로드되는 파일의 내용(Magic Number 등)을 분석하여 차단 여부를 결정합니다.
복합 객체 <comb-obj>, <c-l-1>, <c-l-2> "특정 시간 + 특정 그룹 + 특정 사이트"처럼 여러 조건을 AND/OR로 조합합니다.
정책 실행 <layer>, <rowItem>, <colItem> 최종 실행 단계. 정의된 모든 객체를 조합하여 '누가 어디를 갈 때 어떻게 할지' 결정합니다.

3. 관계성 요약

💡 VPM XML의 핵심 동작 원리

1. 정의부 (<conditionObjects> 등):
모든 재료(대상, 시간, 앱, 확장자 등)를 태그별로 잘게 쪼개어 정의합니다. 특히 <comb-obj>는 작은 태그들을 묶어서 더 큰 조건을 만드는 역할을 합니다.

2. 참조부 (<layer>):
정의된 재료들을 불러와서 실제 규칙(<rowItem>)을 만듭니다. 이때 <colItem> 태그가 앞서 정의된 객체들의 ID나 이름을 참조하여 서로를 연결(Mapping)합니다.

3. 최종 결과:
사용자가 네트워크 접속을 시도하면, <layers>에 있는 규칙들을 최상단부터 순차적으로(Top-Down) 검사하여 조건이 일치하는 규칙의 동작(Action)을 수행합니다.
